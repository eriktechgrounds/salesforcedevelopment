Cascade laat zich vertalen als (kleine) waterval. Zoals het water in een waterval eerst de rotsen bovenaan raakt
en uiteindelijk de laagste - zo wordt in CSS (Cascading Style Sheet) van boven naar beneden geinterpreteerd.
Bij een gelijk hierarchische niveau, komt het laatst (laagst) genoemde element tot uiting.
Inheritance houdt in dat in sommige gevallen, een child-element eigenschappen "erft" (inheritance) van het
parent element. Omdat het child-element per definitie lager geplaatst is dan het parent element, kan
deze georven eigenschap, volgens het cascading principe, dus wel nader gewijzigd worden.

Conflicterende declaraties worden in de volgende volgorde toegepast, waarbij latere declaraties voorrang hebben op eerdere:
  1. Declaraties in user agent stylesheets (bijvoorbeeld de standaardstijlen van de browser, die worden gebruikt als er
    geen andere styling is ingesteld).
  2. Normale declaraties in user style sheets (aangepaste stijlen ingesteld door een gebruiker).
  3. Normale declaraties in author stylesheets (dit zijn de stijlen die door ons, de webontwikkelaars, zijn ingesteld).
  4. !important declaraties in author stylesheets
  5. !important declaraties in user stylesheets
Het is logisch dat de stylesheets van webontwikkelaars de stylesheets van gebruikers overschrijven, zodat het ontwerp kan
blijven zoals het bedoeld is, maar soms hebben gebruikers goede redenen om de stijlen van webontwikkelaars te overschrijven,
zoals hierboven vermeld - dit kan bereikt worden door !important in hun regels te gebruiken.
